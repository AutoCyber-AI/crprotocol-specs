# Contributing to CRP Specifications

Thank you for your interest in the Context Relay Protocol (CRP) specifications.

## How to Contribute

### Issue Reports

We welcome issue reports for:
- Inconsistencies or ambiguities in the spec text
- Technical errors or incorrect normative statements
- Missing cross-references between specs

Please open an issue on this repository with:
1. The spec number (e.g. CRP-SPEC-002)
2. The section and paragraph
3. A clear description of the problem
4. If applicable, a suggested correction

### IETF Feedback

Substantive technical feedback is best directed through the IETF mailing lists:
- HTTP WG: [ietf-http-wg@w3.org](mailto:ietf-http-wg@w3.org)
- IETF Dispatch: [dispatch@ietf.org](mailto:dispatch@ietf.org)

Reference the relevant Internet-Draft(s):
- `draft-vidiniotis-crp-headers`
- `draft-vidiniotis-crp-safety-policy`

### Pull Requests

We do **not** accept pull requests at this time.

The specification is in a pre-standards-track phase. Substantive changes are coordinated through the IETF process. This policy will be revisited upon Working Group formation.
